# Multi-Tenant Cloud

This page describes the security architecture and data protection measures for Onyx's multi-tenant cloud offering.

## Multi-Tenant Architecture

### Data Isolation
- **Logical Separation**: Complete logical separation of tenant data
- **Encryption**: Tenant-specific encryption keys for all data
- **Access Controls**: Strict access controls preventing cross-tenant data access
- **Network Isolation**: Network-level isolation between tenants

### Shared Infrastructure
- **Compute Resources**: Shared compute resources with tenant isolation
- **Storage**: Multi-tenant storage with logical separation
- **Network**: Shared network infrastructure with security controls
- **Monitoring**: Tenant-aware monitoring and alerting

## Security Measures

### Data Protection
- **Encryption at Rest**: AES-256 encryption with tenant-specific keys
- **Encryption in Transit**: TLS 1.3 for all data transmission
- **Access Logging**: Comprehensive audit trails for all data access
- **Data Backup**: Encrypted backups with tenant isolation

### Access Controls
- **Authentication**: Multi-factor authentication for all users
- **Authorization**: Role-based access controls with tenant boundaries
- **Session Management**: Secure session handling with tenant isolation
- **API Security**: Tenant-aware API access controls

## Compliance and Certifications

### Security Certifications
- **SOC 2 Type II**: Annual security audits and certifications
- **ISO 27001**: Information security management system certification
- **GDPR Compliance**: Full GDPR compliance for EU data processing
- **Regular Audits**: Third-party security audits and penetration testing

### Data Residency
- **Regional Deployment**: Data stored in your chosen region
- **Compliance Support**: Support for various compliance frameworks
- **Data Sovereignty**: Respect for local data sovereignty requirements
- **Audit Support**: Support for customer security audits
