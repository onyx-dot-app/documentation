A Craft **session** is your workspace for one piece of work. It holds the conversation, the files you bring in,
the files Craft creates as it works, and the finished artifacts.
This page walks through a session from first prompt to shareable result.

<img className="rounded-image" src="/assets/overview/core_features/craft_workspace.png" alt="A Craft session: sidebar, chat panel, and the Output panel previewing a generated web app"/>

## The session workspace

A Craft session has three areas:

- **Sidebar** (left) — your session history and links to Scheduled Tasks, Skills, and Apps. Start a new session with
  **Start Crafting**.
- **Chat panel** (center) — where you talk to Craft, watch it work, and respond to approval prompts.
- **Output panel** (right) — where you preview web apps, browse files, and download artifacts. Toggle it with the
  panel button in the session header.

## Write a strong first prompt

Before you send your first message, pick a model if your workspace offers more than one.
The model is **locked once the session has messages**, so choose before you begin.

Craft does its best work when your first prompt covers:

- **The artifact** — a web app, report, deck, brief, data explorer, or export file.
- **The audience and purpose** — who will use it and what for.
- **The source material** — Onyx knowledge, attached files, User Library files, or a connected app.
- **The requirements that matter** — sections, tone, charts, filters, tables, file names, or output format.

<Tip>
  Craft generally won't stop to ask clarifying questions — it makes reasonable assumptions and keeps going.
  Front-load the details you care about, then refine from there.
</Tip>

## Add context

Use the **+** button in the input bar (**Add files or skills**) or type **/** to bring in context.
Craft can combine several sources in one session.

| Source | Use it for |
| --- | --- |
| Onyx knowledge | Company documents and connector data you already have permission to access. |
| Session attachments | Files that only matter for this session — a CSV, PDF, notes doc, or image. |
| User Library | Reusable files you want available across many sessions, like templates or datasets. |
| Skills | A specific method or capability, such as building a deck or searching company knowledge. |
| Apps | External services such as Slack, Google Drive, Gmail, Linear, or GitHub. |

Session attachments are tied to one session. User Library files are durable and reusable.
You can also drag and drop files directly onto the chat.

<Accordion title="File and storage limits">
  These are the default limits. Your admin can adjust them for self-hosted deployments.

  **Session attachments**

  | Limit | Default |
  | --- | --- |
  | Maximum file size | 50 MB |
  | Maximum files per session | 20 |
  | Maximum total size per session | 200 MB |

  **User Library**

  | Limit | Default |
  | --- | --- |
  | Maximum file size | 500 MB |
  | Maximum files per upload | 100 |
  | Maximum total storage | 10 GB |

  The User Library accepts regular files and zip uploads. Zips are extracted so Craft can work with the files inside.
</Accordion>

<Warning>
  Don't upload secrets, private tokens, or production credentials as files. To let Craft reach an external service,
  connect it as an [app](/overview/core_features/craft_skills_and_apps#apps)
  instead — credentials are injected securely and never stored in the sandbox.
</Warning>

## While Craft works

As Craft runs, the chat panel streams progress: task lists, generated files, tool results, approval prompts,
and any errors. A status indicator in the header shows the sandbox state (for example, *Initializing sandbox…*,
*Sandbox running*, or *Restoring sandbox…*).

- **Queue a follow-up** — type and send another message while a run is active. Craft picks it up after the current
  turn finishes.
- **Interrupt** — press **`Esc`** to stop a run that's going the wrong way. Your messages and files are kept, and you
  can send a new prompt right after.
- **Subagents** — for complex work, Craft may spin up subagents for focused subtasks. Switch to a subagent from the
  header dropdown to read its transcript; your replies still go to the main session.

## The Output panel

The Output panel has three tabs:

| Tab | Shows |
| --- | --- |
| **Preview** | Your live web app. Ask Craft for changes and the preview updates as the files change. |
| **Files** | The full session workspace — source code, data, and intermediate work. |
| **Artifacts** | Finished outputs meant for download or sharing, usually written to `outputs/`. |

<img className="rounded-image" src="/assets/overview/core_features/craft_files.png" alt="The Output panel showing the Preview, Files, and Artifacts tabs with the session workspace files"/>

From here you can:

- Download an individual file, or a directory or web app as a zip.
- Export a Markdown file to **.docx**.
- Preview supported images, PDFs, presentations, and text files.
- Share a generated web app (see below).

## Share a web app

Generated web apps are previewed and shared through Onyx, not exposed directly from the sandbox.
Use the **Share** button on the Preview tab to choose a scope:

- **Private** — only you can view the app.
- **Organization** — anyone logged into your Onyx can view it.

Sharing covers only the generated web app. It does **not** share the conversation or the rest of your session files.

## Iterate

Craft is built for back-and-forth. Once you have a first draft, refine it in plain language:

- "Add a filter for the last 30 days and move the summary cards to the top."
- "Rewrite the executive summary for a non-technical audience and cut it to three sentences."
- "Export the final report to DOCX and put it in `outputs/`."

## Troubleshooting

| Issue | What to try |
| --- | --- |
| Craft isn't in my sidebar | Ask an admin whether Craft is enabled and a supported model provider is configured. |
| Craft can't read a file | Check the [file limits](#add-context), try a simpler format, or move reusable files to the User Library. |
| Preview is blank or broken | Ask Craft to inspect the app error. If the sandbox is restoring, wait a moment and retry. |
| A run is stuck or off track | Press **`Esc`** to interrupt, then send a more specific prompt. |
| An action is waiting for approval | Review the [approval card](/overview/core_features/craft_skills_and_apps#approvals) and approve only if it matches your intent. Approvals expire if left unanswered. |
| I can't find a generated file | Ask Craft to place final files in `outputs/` with clear names. |
