# Configuration Overview

Onyx is highly configurable to meet your organization's specific needs. Understanding how it's set up helps you get the most out of the platform.

## Data Sources & Connectors

Your Onyx instance connects to various data sources through connectors:

### **Common Connectors**
- **Documentation**: Confluence, Notion, SharePoint
- **Code Repositories**: GitHub, GitLab, Bitbucket
- **Communication**: Slack, Microsoft Teams
- **Files**: Google Drive, Dropbox, OneDrive
- **Databases**: PostgreSQL, MySQL, MongoDB

### **How It Works**
Connectors automatically sync data from your sources, making it searchable through Onyx. The sync frequency and scope are configured by your administrators.

## AI Agents

Your organization can customize AI agents for different purposes:

### **Specialized Agents**
- **Department-specific**: HR, Engineering, Sales agents
- **Role-based**: Manager, Developer, Support agents  
- **Project-focused**: Product launch, compliance agents

### **Agent Configuration**
- Configure agent personalities, knowledge bases, and capabilities
- Set access controls and permissions
- Define custom tools and integrations

## Search Configuration

### **Search Capabilities**
- **Semantic Search**: Understanding meaning and context
- **Keyword Search**: Traditional text matching
- **Hybrid Search**: Combining both approaches
- **Faceted Search**: Filtering by source, date, type

### **Ranking and Relevance**
Search results are optimized based on:
- Document relevance and authority
- User permissions and access rights
- Freshness and update frequency
- User interaction patterns

## Access Control

### **User Management**
- **Authentication**: SSO, LDAP, or local accounts
- **Authorization**: Role-based access control
- **Permissions**: Document-level and source-level restrictions

### **Group-Based Access**
- **Teams**: Department or project-based groups
- **Roles**: Admin, user, viewer permissions
- **Resources**: Controlled access to specific agents and data

### **Workflow Integration**

### **Analytics**

### **Data Protection**
