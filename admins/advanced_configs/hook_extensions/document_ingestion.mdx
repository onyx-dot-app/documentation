## Overview

The Document Ingestion hook point lets you intercept every document before it enters the Onyx indexing pipeline. Your endpoint receives the fully-formed document and can filter it out, rewrite its content, or pass it through unchanged.

It runs immediately after Onyx's internal validation and before the indexing pipeline begins — no partial writes have occurred yet.

## Defaults

| Setting | Value |
|---|---|
| Default Timeout | 30 seconds (configurable) |
| Default Fail Strategy | Hard (configurable) |
| On Hard Fail | The document will not be indexed |

## Input Schema

Onyx sends a `POST` request to your endpoint once per document with the following JSON body:

```python
class DocumentIngestionSection(BaseModel):
    text: str | None = None          # set for text sections, null for image sections
    link: str | None = None
    image_file_id: str | None = None # set for image sections (opaque id), null for text sections

class DocumentIngestionOwner(BaseModel):
    display_name: str | None = None
    email: str | None = None

class DocumentIngestionPayload(BaseModel):
    document_id: str
    title: str | None
    semantic_identifier: str
    source: str
    sections: list[DocumentIngestionSection]
    metadata: dict[str, list[str]]
    doc_updated_at: str | None
    primary_owners: list[DocumentIngestionOwner] | None
    secondary_owners: list[DocumentIngestionOwner] | None
```

### DocumentIngestionPayload

| Field | Type | Required | Description |
|---|---|---|---|
| `document_id` | string | Yes | Unique identifier for the document. **Read-only** — changes are ignored. |
| `title` | string \| null | Yes | Title of the document. |
| `semantic_identifier` | string | Yes | Human-readable identifier used for display (e.g. file name, page title). |
| `source` | string | Yes | Connector source type (e.g. `confluence`, `slack`, `google_drive`). **Read-only** — changes are ignored. |
| `sections` | DocumentIngestionSection[] | Yes | All sections of the document — both text sections and image sections. |
| `metadata` | dict | Yes | Key-value metadata. Values are always `string[]`. |
| `doc_updated_at` | string \| null | Yes | ISO 8601 timestamp of the last update at the source, or `null` if unknown. |
| `primary_owners` | DocumentIngestionOwner[] \| null | Yes | Primary owners of the document, or `null` if not available. |
| `secondary_owners` | DocumentIngestionOwner[] \| null | Yes | Secondary owners of the document, or `null` if not available. |

### DocumentIngestionSection

| Field | Type | Required | Description |
|---|---|---|---|
| `text` | string \| null | Yes | Text content. Set for text sections, `null` for image sections. |
| `link` | string \| null | No | Optional URL associated with this section. |
| `image_file_id` | string \| null | No | Opaque image identifier. Set for image sections, `null` for text sections. Image content is not included — hooks can reorder or drop image sections but cannot read or modify the image. |

### DocumentIngestionOwner

| Field | Type | Required | Description |
|---|---|---|---|
| `display_name` | string \| null | No | Human-readable name of the owner. |
| `email` | string \| null | No | Email address of the owner. |

### Example

```json
{
  "document_id": "confluence::page::123456",
  "title": "Q4 2024 Refund Policy",
  "semantic_identifier": "Q4 2024 Refund Policy",
  "source": "confluence",
  "sections": [
    {
      "text": "Our refund policy allows returns within 30 days...",
      "link": "https://wiki.example.com/pages/123456",
      "image_file_id": null
    },
    {
      "text": null,
      "link": null,
      "image_file_id": "846d6333-1a2b-4c5d-8e9f-0a1b2c3d4e5f"
    }
  ],
  "metadata": {
    "space": ["HR"],
    "labels": ["policy", "finance"]
  },
  "doc_updated_at": "2024-10-01T12:00:00Z",
  "primary_owners": [
    { "display_name": "Alice Smith", "email": "alice@example.com" }
  ],
  "secondary_owners": null
}
```

## Output Schema

A successful response from your endpoint must return HTTP `200` and a JSON body:

| Field | Type | Required | Description |
|---|---|---|---|
| `sections` | DocumentIngestionSection[] \| null | Yes | The sections to index in the desired order. The hook controls final ordering. `null` or empty list drops the document. |
| `rejection_reason` | string \| null | No | Logged when `sections` is `null` or empty. Falls back to a generic message if omitted. |

### Pass through unchanged

```json
{
  "sections": [
    {
      "text": "Our refund policy allows returns within 30 days...",
      "link": "https://wiki.example.com/pages/123456",
      "image_file_id": null
    },
    {
      "text": null,
      "link": null,
      "image_file_id": "a1b2c3d4-e5f6-7890-abcd-ef1234567890"
    }
  ]
}
```

### Rewrite content (PII redaction)

```json
{
  "sections": [
    { "text": "Our refund policy allows returns within 30 days. Contact [REDACTED] for details." }
  ]
}
```

### Drop the document

```json
{
  "sections": null,
  "rejection_reason": "Document contains restricted classification label."
}
```

## Authentication

If you configured an API key when registering the hook, Onyx includes it in every request:

```
Authorization: Bearer <your-api-key>
```
